Учебный defensive-инструмент для Windows, PowerShell и практической диагностики.

Windows Security Workbench связывает команды, сценарии диагностики, live-проверки, системные артефакты и учебный roadmap в одном приложении. Это не просто каталог команд, а объясняющий тренажёр, который показывает куда смотреть, чем проверять и как читать результат.

Не просто справочник команд

Проект сделан как учебная рабочая станция для defensive-анализа Windows-хоста и параллельного изучения Linux-команд через практику.

Команды

Каталог Windows и Linux команд с назначением, примером вывода, интерпретацией и следующими шагами диагностики.

  • PowerShell, CMD и Linux shell рядом.
  • Фокус на реальной диагностике, а не на теории ради теории.
  • Подходит и как шпаргалка, и как учебный слой.

Практика

Готовые плейбуки по категориям: сеть, диски, процессы, службы, автозагрузка, журналы и доступ.

  • Формат: симптом - что проверить - какие команды - как читать.
  • Удобно для демонстрации преподавателю и для самообучения.
  • Есть сценарии и для Windows, и для Linux basics.

Live-проверки

Безопасное выполнение read-only команд прямо из приложения: Windows через PowerShell, Linux через WSL.

  • Никаких offensive-функций и скрытых действий.
  • Результаты можно экспортировать в Markdown и JSON.
  • История запусков хранится локально.

Что умеет Workbench

Инструмент покрывает диагностические и учебные сценарии, которые реально нужны на Windows-хосте и в защитной практике.

Windows Audit

Проверка учётных записей, сети, firewall, обновлений, служб, задач и системных артефактов.

Safe Cleanup

Безопасная очистка временных файлов и корзины с режимом dry-run.

GUI Toolkit

Тёмная тема, быстрые сценарии, вкладки, экспорт и режим учебного тренажёра по умолчанию.

CLI Scenarios

Режимы audit, cleanup, learn, study, practice, full.

Command Catalog

Windows и Linux команды с пояснениями, примерами вывода и заметками по интерпретации.

Artifacts Map

Справочник: где лежат журналы, задачи, startup locations, firewall logs, registry points и другие артефакты.

WSL Live

Если установлен WSL, проект умеет выполнять Linux read-only проверки для учебной параллели с Windows.

Reports

Экспорт отчётов в Markdown и JSON плюс локальная история запусков.

GUI как учебный стенд

GUI подходит для демонстрации проекта на защите и для повседневной работы как тренажёр.

  • Вкладки: сводка, находки, команды, практика, где искать, обучение, JSON, инструменты.
  • Быстрые сценарии в боковой панели.
  • Запуск в учебном режиме по умолчанию, а не в максимально тяжёлом полном аудите.

CLI как практическая утилита

CLI закрывает тот же набор сценариев, но удобен для отчётов, повторяемых запусков и демонстрации командной части проекта.

  • Скрипт Start-Workbench.ps1 собирает проект при необходимости и запускает нужный режим.
  • Результаты можно сразу выгрузить в файлы отчётов.
  • Подходит и для учебных прогонов, и для мини-аудита локальной машины.

Как запускать

Самый практичный путь для пользователя — launcher, но ручная сборка и Docker cross-build тоже поддержаны.

Быстрый старт

.\Start-Workbench.ps1

Скрипт сам проверяет конфигурацию CMake, при необходимости собирает Release-бинарники и запускает GUI.

Ручная сборка

cmake -S . -B build
cmake --build build --config Release

.\build\Release\WindowsSecurityWorkbenchGUI.exe
.\build\Release\WindowsSecurityWorkbench.exe help

Docker

docker build -t windows-security-workbench:cross .
docker run --rm windows-security-workbench:cross

Контейнер не запускает GUI внутри Linux. Он кросс-компилирует Windows .exe через mingw-w64.

Режимы CLI

GUI и CLI используют общие учебные сценарии, поэтому переключение между режимами не ломает модель работы.

audit

Defensive-аудит Windows-хоста с учебной подачей результатов и возможным экспортом отчёта.

cleanup

Очистка кэша, временных файлов и корзины. Для безопасного прогона используется --dry-run.

learn

Команды, плейбуки, инструменты, артефакты и учебный roadmap в одном режиме.

study

Отдельный учебный слой без полноценного аудита и без акцента на действия с системой.

practice

Практический режим с live-проверками Windows и, при желании, Linux через --linux-live.

full

Комбинированный сценарий: аудит, очистка и учебные материалы в одном прогоне.

Границы безопасности

Проект намеренно ограничен defensive-only сценарием и не уходит в offensive-плоскость.

  • Не извлекает пароли браузеров.
  • Не получает Wi-Fi ключи, токены, хэши и другие секреты.
  • Не работает с LSASS и не делает credential dumping.
  • Live-режим запускает только безопасные read-only проверки.

Почему проект удобен для диплома

Это понятная и демонстрируемая тема: есть GUI, CLI, архитектура по модулям, экспорт, Docker cross-build и реальная учебная польза.

  • Нормально смотрится в GitHub-портфолио.
  • Можно показывать и архитектуру, и интерфейс, и прикладной результат.
  • Есть понятный путь развития: поиск, фильтры, SQLite, история, дифф запусков.

Как показать проект преподавателю

Ниже самый практичный сценарий демонстрации без лишней постановочности.

Запустить launcher Показать, что проект стартует одной командой через .\Start-Workbench.ps1.
Открыть практический режим Переключиться на категории вроде сети, дисков, автозагрузки и журналов.
Выполнить live-проверки Windows Показать, что проект умеет запускать только безопасные read-only команды внутри приложения.
Экспортировать результаты Сохранить отчёты в Markdown и JSON.
Показать Linux-слой Объяснить, что Linux-команды и live-проверки доступны через WSL, если он установлен.
